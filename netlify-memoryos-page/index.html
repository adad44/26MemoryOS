What it is

A memory layer for your own computer.

MemoryOS now ships as a local macOS system: Swift capture daemon, Swift menu bar app, Chrome extension, local FastAPI backend, React web UI, SQLite database, search index, storage controls, todos, and a local You user-model tab.

Capture

Records useful context from active windows, browser pages, and local files you already use.

Search

Builds a local index with TF-IDF by default, with hooks for sentence-transformer and FAISS search.

Review

Search, pin, label, batch-review, inspect collections, read weekly digests, and create todos.

Control

Pause capture from the menu bar, manage privacy lists, export JSON, clean storage, or delete local data.

Why it matters

Most useful context disappears before you know you need it.

A bug trace, a class portal page, a repo file, a browser tab, or a half-remembered note can be hard to recover later. MemoryOS gives you a private recall layer based on your actual workflow, not a blank prompt.

1

Find the file, tab, command, page, or app state you were using without reconstructing your day.

2

Keep noisy captures out of the index by labeling them directly in the UI.

3

Create a local dataset for better personal search models from your own labeled activity.

AI agent memory

Your coding agent can ask MemoryOS what you already did.

MemoryOS exposes a localhost FastAPI surface so tools like Codex, Claude Code, Cursor, or another desktop agent can search your local captured context instead of asking you to reconstruct it from memory.

Search tool

Agents can call POST /search to retrieve matching local captures, notes, pages, files, and debug context.

Recent context

GET /recent lets an agent inspect what you were just working on without touching raw SQLite.

User model

GET /user-model and GET /beliefs expose local project interests, patterns, and knowledge gaps.

Safe actions

Open, todo, cleanup, and delete endpoints exist, but destructive actions should stay confirmation-gated.

Local ML search

Start lightweight, then switch on local models when you need them.

The default installer uses TF-IDF so search works immediately on a normal Mac. If you want semantic search, install the embedding extras and rebuild the index with a local sentence-transformer and optional FAISS backend.

1

Use the default TF-IDF index for fast local search with no model download.

2

Run scripts/install_memoryos.sh --with-embeddings to enable sentence-transformer and FAISS search experiments.

3

Use the local Ollama-backed user model to summarize recurring projects, interests, and patterns from captured evidence.

Can you trust it?

Screen-reading software has to earn trust first.

MemoryOS is designed to be inspectable before it is automatic. It keeps data local, exposes the capture database, and makes privacy controls part of the product instead of a footnote.

✓

Data is stored in a local SQLite database on your Mac.

✓

No hosted account, subscription, payment flow, or cloud sync is required.

✓

You can inspect captures, label noise, export records, and delete local data.

✓

Sensitive apps and paths can be blocked through privacy settings.

How to get it

Install the full local system with one command.

This command downloads MemoryOS and runs the installer script for you. The installer creates the Python environment, installs backend and web dependencies, builds the React UI, installs or starts Ollama, pulls the local model unless skipped, builds the Swift daemon and menu bar app, registers launch agents, and opens the web UI.

curl -fsSL https://memoryos-mac.netlify.app/install.sh | bash
1

Run the install command once from Terminal.

2

Grant Accessibility and Disk Access permissions when prompted.

3

Use the web UI to search your history, label captures, and view your personal memory stats.
